# Ember & Thread

A cozy top-down pixel-art adventure about mending broken magical objects,
recovering lost memories, and learning to understand someone who experiences
the world differently.

Built in Godot 4, by one solo creator, as a gift. Cozy first, always.

## Running the game

1. Install [Godot 4.4 or newer](https://godotengine.org/download) (the
   standard build — no .NET needed).
2. In Godot's Project Manager, choose **Import** and select this folder's
   `project.godot`.
3. Press <kbd>F5</kbd> to run.

The first time the editor opens the project it generates `*.import` and
`*.uid` files next to assets and scripts — commit those. The `.godot/`
cache folder stays git-ignored.

## Controls

| Action | Keyboard | Controller |
|---|---|---|
| Move | WASD / arrow keys | Left stick / D-pad |
| Interact *(reserved)* | E / Space / left click | A |
| Journal *(reserved)* | J / Tab | Y |
| Comfort pouch *(reserved)* | C | X |
| Pause *(reserved)* | Esc | Start |

Bindings use physical key positions, so WASD works on any keyboard layout.
*Reserved* actions are defined in the input map but nothing consumes them
yet. Remappable controls arrive with the settings menu (accessibility
requirement).

## Project structure

```
assets/        locked asset tree (art-pipeline.md §3) — stand-ins live at
               the exact final path real art will overwrite 1:1
core/          the pipeline seams: AssetRegistry, SoundManager, MusicManager
docs/          source-of-truth documents
scenes/        one folder per feature; scene + script + README together
tools/         make_placeholders.py — regenerates block stand-ins + manifest
```

## Art pipeline (short version)

Art is a swappable layer — see `docs/art-pipeline.md` for the full rules.

- Systems reference art by logical key through `AssetRegistry`, never by
  path. Real art overwrites a stand-in file in place; zero code edits.
- Block stand-ins are magenta-marked and regenerated by
  `python3 tools/make_placeholders.py`; debug builds list every stand-in
  still in use at startup (hash-checked via `assets/stand_ins.json`).
- Gap-filler sourcing is **Kenney only** (CC0), conformed to the locked
  sizes/naming and tracked in `assets/temp_assets.md`. The player, Pip,
  NPCs, portraits, journal illustrations and the map are never sourced —
  blocks until the artist delivers.
- Raw Kenney packs live at the repo root (`Roguelike Base Pack/`,
  `Roguelike Characters Pack/` — 16×16 cells, 1px margins, `.gdignore`d so
  Godot never imports them). Slice cells into conformed assets with
  `python3 tools/conform_kenney.py base COL ROW tilesets/coastal/tile_… .png`
  — it stitches multi-cell blocks (`--cells 2x2`), integer-scales
  (`--scale`), and records the row in `temp_assets.md` automatically.

## Pixel-art display settings (and why)

- Native resolution **480×270**, exactly 4× at 1920×1080.
- Stretch mode `canvas_items` with integer scaling and nearest filtering:
  the world stays on the chunky pixel grid while text/UI render at window
  resolution — which is what makes adjustable text size achievable
  (accessibility requirement).
- `snap_2d_transforms_to_pixel` keeps sprites aligned to the art grid.
